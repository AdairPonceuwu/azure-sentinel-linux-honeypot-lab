# Attack Map

## Purpose

This document explains how the SSH attack map was created in Microsoft Sentinel.

The map visualizes public source IP addresses that attempted to authenticate to the Debian SSH honeypot.

## Data Source

The map uses Linux SSH authentication logs stored in the `Syslog` table.

Relevant log patterns:

- `Invalid user`
- `Connection closed by invalid user`
- `Connection reset by invalid user`
- `Failed password`

## Geolocation Approach

The original Windows-based lab used a Microsoft Sentinel Watchlist with a GeoIP CSV file.

In this implementation, Watchlist creation was not available, so IP geolocation enrichment was performed directly in KQL using:

```kql
geo_info_from_ip_address()
```

This allowed the lab to continue without importing an external GeoIP CSV file.

## Attack Map Query

The complete query is stored in:

```text
kql/ssh/ssh-geoip-attack-map.kql
```

The query performs these steps:

1. Reads SSH authentication events from the `Syslog` table.
2. Filters events generated by `sshd`.
3. Extracts source IP addresses from `SyslogMessage`.
4. Uses `geo_info_from_ip_address()` to enrich IP addresses.
5. Produces latitude and longitude fields.
6. Summarizes event count by attacker IP and location.
7. Outputs the fields required by the Workbook map visualization.

## Required Output Fields

The Workbook map expects these fields:

| Field | Purpose |
|---|---|
| `FailureCount` | Number of observed attempts |
| `AttackerIp` | Source IP address |
| `latitude` | Map latitude |
| `longitude` | Map longitude |
| `city` | Source city, when available |
| `country` | Source country, when available |
| `friendly_location` | Label shown in the map |

## Workbook

The Workbook JSON should be stored in:

```text
workbooks/ssh-attack-map-workbook.json
```

## Workbook Visualization Settings

Recommended map settings:

| Setting | Value |
|---|---|
| Visualization | Map |
| Location info | Latitude / Longitude |
| Latitude column | `latitude` |
| Longitude column | `longitude` |
| Size column | `FailureCount` |
| Label column | `friendly_location` |
| Legend metric | `FailureCount` |

## Results

The final map displayed source locations for SSH authentication attempts against the honeypot. Marker size was based on the number of observed attempts from each source.

Geolocation data may not always be precise. The map should be interpreted as an enrichment and visualization tool, not as definitive attribution.
